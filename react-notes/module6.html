📝 How to use this guide: React by default is a Single Page Application (SPA), meaning it doesn't have multiple pages. This module teaches you how to use React Router to fake the illusion of multiple pages. Pay close attention to Protected Routes and Dynamic Parameters, as these are foundational for any enterprise dashboard.

1. React Router DOM (v6/v7) Setup

1. Definition

`react-router-dom` is the standard library for routing in React web applications. It keeps your UI synchronized with the browser's URL, enabling navigation without page reloads.

2. Explanation

  • In traditional websites, navigating to a new URL sends a request to the server, which returns a completely new HTML page.
  • React Router intercepts URL changes, prevents the browser request, and simply swaps out the React components currently rendered on the screen.
  • Modern setups (v6.4+) highly encourage using the "Data Router" (createBrowserRouter) over the older <BrowserRouter> component wrapper.

3. Example Code

import { createBrowserRouter, RouterProvider } from 'react-router-dom';
import Home from './pages/Home';
import About from './pages/About';

// 1. Define the routing tree
const router = createBrowserRouter([
  { path: '/', element: <Home /> },
  { path: '/about', element: <About /> },
]);

// 2. Provide the router to the application root
const App = () => {
  return <RouterProvider router={router} />;
};

4. Real-Time Scenarios & Usages

  • Any application that has more than one "view" (e.g., Home page, Login page, Dashboard, User Settings).

5. Advantages & Disadvantages

Advantages: The absolute industry standard. Enormous ecosystem and documentation. Enables deep-linking and browser history navigation (back/forward buttons).
Disadvantages: The API changes drastically between major versions (v5 to v6 was a massive rewrite), causing significant migration headaches for legacy apps.

⚡ Interview Corner

Q: What is the difference between `BrowserRouter` and `HashRouter`?
A: BrowserRouter uses the modern HTML5 History API to create clean URLs (e.g., domain.com/about). It requires server-side configuration to route all requests back to index.html. HashRouter uses the hash portion of the URL (e.g., domain.com/#/about). It doesn't require server configuration because everything after the # is ignored by the server, making it useful for simple static file hosts.

2. Routes, Route, Link, NavLink

1. Definition

These are the core components of React Router. `Routes` groups paths, `Route` maps a path to a component, `Link` allows navigation, and `NavLink` is a special link that knows if it is "active".

2. Explanation

  • <Routes> looks through all its child routes to find the best match for the current URL.
  • <Route> pairs a specific path string to an element (JSX).
  • You must NEVER use standard <a href=""> tags to navigate internally, as they force a full browser refresh. Use <Link to=""> instead.
  • <NavLink> is just a <Link> that provides an isActive boolean to its className or style props so you can highlight the current page in a navbar.

3. Example Code

import { Routes, Route, Link, NavLink } from 'react-router-dom';

const Navbar = () => (
  <nav>
    <Link to="/">Brand Logo</Link>
    
    {/* NavLink automatically applies the 'active' class when on /dashboard */}
    <NavLink 
      to="/dashboard"
      className={({ isActive }) => isActive ? "text-blue-500 font-bold" : "text-gray-500"}
    >
      Dashboard
    </NavLink>
  </nav>
);

4. Real-Time Scenarios & Usages

  • Using NavLink to build top navigation bars or sidebars where the currently selected menu item needs to be highlighted.
  • Using Link to build footer links or "Read More" buttons on blog cards.

5. Advantages & Disadvantages

Advantages: Declarative approach to routing. Links are accessible out of the box.
Disadvantages: None. This is the fundamental way to navigate in a React SPA.

⚡ Interview Corner

Q: Why is using `<a href="">` bad inside a React application?
A: An <a href> tag completely discards the current web page and asks the server for a new one. This triggers a full page reload, wiping out the entire React application state, Context, and Redux store. <Link> prevents this default behavior and uses the browser's History API to change the URL without reloading the page.

3. Dynamic Routing (useParams) & Query Parameters

1. Definition

Dynamic Routing allows you to capture values directly from the URL. `useParams` reads dynamic path segments (e.g., `/user/:id`), while `useSearchParams` reads the query string (e.g., `?sort=asc`).

2. Explanation

  • Path Parameters: Defined in the route config using a colon (e.g., path="/profile/:username"). Extracted using const { username } = useParams(). Used for identifying specific resources.
  • Search Parameters: The string after the ? in a URL. Extracted using const [searchParams, setSearchParams] = useSearchParams(). It behaves exactly like useState, but stores the state inside the URL.

3. Example Code

import { useParams, useSearchParams } from 'react-router-dom';

const ProductPage = () => {
  // Extracting from URL path: e.g., /products/123
  const { productId } = useParams(); 
  
  // Extracting from Query String: e.g., /products/123?color=red
  const [searchParams, setSearchParams] = useSearchParams();
  const color = searchParams.get('color') || 'blue';

  const changeColor = (newColor) => {
    // Updates URL to ?color=newColor without reloading
    setSearchParams({ color: newColor }); 
  };

  return <div>Viewing Product #{productId} in {color}</div>;
};

4. Real-Time Scenarios & Usages

  • useParams: Detail pages. Clicking a user profile goes to /users/dave123. The component uses useParams to get "dave123" and fetches Dave's data from the API.
  • useSearchParams: Filters, sorting, and pagination on a grid. Storing these in the URL means the user can bookmark the page or share the link with the exact filters applied.

5. Advantages & Disadvantages

Advantages: Makes UI state shareable and bookmarkable (URL-driven state).
Disadvantages: URL Search Params are strictly strings. You must manually parse booleans or numbers (e.g., Number(searchParams.get('page'))).

⚡ Interview Corner

Q: When should you store state in `useState` versus `useSearchParams` (the URL)?
A: You should store state in the URL if it dictates what the user is currently looking at and they might want to share or bookmark that exact view (e.g., search queries, active tab, page number). You should use useState for ephemeral UI state that doesn't matter if it's lost on refresh (e.g., a dropdown menu being open or closed, form inputs before submission).

4. Programmatic Navigation (useNavigate)

1. Definition

`useNavigate` is a hook that allows you to trigger URL changes via JavaScript code, rather than waiting for the user to physically click a `` component.

2. Explanation

  • It returns a function (typically named navigate).
  • You can pass a string path to navigate somewhere new: navigate('/dashboard').
  • You can pass a number to go forward or backward in the browser history stack: navigate(-1) acts like clicking the browser's "Back" button.
  • You can pass options like { replace: true } to overwrite the current history entry.

3. Example Code

import { useNavigate } from 'react-router-dom';

const LoginForm = () => {
  const navigate = useNavigate();

  const handleLogin = async (e) => {
    e.preventDefault();
    const success = await loginUser(); // some API call
    
    if (success) {
      // Redirect to dashboard on success, replacing the login page in history
      navigate('/dashboard', { replace: true });
    }
  };

  return (
    <form onSubmit={handleLogin}>
      {/* form fields... */}
      <button type="submit">Log In</button>
    </form>
  );
};

4. Real-Time Scenarios & Usages

  • Redirecting a user after a successful login or form submission.
  • Building a "Cancel" or "Go Back" button.
  • Redirecting a user away from a page if they lack permissions, based on an API response.

5. Advantages & Disadvantages

Advantages: Essential for handling business logic that dictates where the user should go based on data outcomes.
Disadvantages: Should not be used to replace simple <Link> tags. Putting an onClick={navigate} on a standard <div> instead of using an anchor breaks accessibility and SEO.

⚡ Interview Corner

Q: What exactly does `Maps('/home', { replace: true })` do?
A: Normally, navigating pushes a new entry onto the browser's history stack. If the user clicks the browser's "Back" button, they go to the previous page. The replace: true option replaces the current entry in the history stack instead. This is crucial for login pages—after logging in, you replace the login route so that clicking "Back" doesn't take the user back to the login screen.

5. Protected Routes (Authentication wrappers)

1. Definition

A Protected Route is a structural pattern where a route is wrapped by a higher-order component that checks the user's authentication status before allowing them to view the page.

2. Explanation

  • React Router doesn't have built-in "auth" features. You have to build it yourself.
  • You create a custom component (e.g., <ProtectedRoute>) that checks global state (Context/Redux) to see if the user is logged in.
  • If logged in, it renders the child components.
  • If NOT logged in, it immediately redirects the user to the /login route using the <Navigate /> component.

3. Example Code

import { Navigate } from 'react-router-dom';
import { useAuth } from './hooks/useAuth';

const ProtectedRoute = ({ children }) => {
  const { isAuthenticated, isLoading } = useAuth();

  if (isLoading) return <div>Loading...</div>; // Prevent premature redirect

  if (!isAuthenticated) {
    // Navigate immediately without user interaction
    return <Navigate to="/login" replace />;
  }

  return children;
};

// Usage in router setup:
// <Route path="/dashboard" element={
//   <ProtectedRoute> <Dashboard /> </ProtectedRoute>
// } />

4. Real-Time Scenarios & Usages

  • Securing admin panels, user profile settings, and payment portals so unauthenticated users cannot access them by manually typing the URL.

5. Advantages & Disadvantages

Advantages: Centralizes security logic. You don't have to write "check if user is logged in" at the top of every single protected component.
Disadvantages: Client-side route protection is fundamentally insecure. An attacker can still download the JS bundle. True security must always happen at the API layer (Backend).

⚡ Interview Corner

Q: How do you redirect a user back to their intended page after they finish logging in?
A: When the ProtectedRoute redirects the user to /login, it should pass the current location inside the state prop: <Navigate to="/login" state={{ from: location.pathname }} />. Then, inside the Login component, after a successful login, you read that state and redirect them back: navigate(state?.from || '/dashboard').

6. Layout Routes and Nested Routing (<Outlet />)

1. Definition

Nested routing allows you to define child routes inside parent routes. The `` component is placed in the parent component and acts as a placeholder where the matched child component will be injected.

2. Explanation

  • Often, multiple pages share the exact same layout (e.g., a Sidebar and a Top Navbar).
  • Instead of importing the Sidebar into every single page, you create a "Layout Component".
  • You make the pages "children" of this Layout Route in the router configuration.
  • The Layout component renders the persistent UI (Sidebar) and an <Outlet />. React Router injects the correct page into the Outlet based on the URL.

3. Example Code

import { Outlet } from 'react-router-dom';

// 1. The Layout Component
const DashboardLayout = () => {
  return (
    <div className="flex">
      <Sidebar /> {/* This persists across all child routes! */}
      <div className="main-content">
        <Outlet /> {/* Child route components are injected here */}
      </div>
    </div>
  );
};

// 2. Router Config
/*
  <Route path="/app" element={<DashboardLayout />}>
    <Route path="profile" element={<UserProfile />} /> 
    <Route path="settings" element={<Settings />} />
  </Route>
*/

4. Real-Time Scenarios & Usages

  • Dashboard architectures where a left sidebar stays on screen while the right-side content changes.
  • Multi-step checkout flows where the header/footer and progress bar remain constant.

5. Advantages & Disadvantages

Advantages: Drastically improves performance. When navigating between child routes, the Layout component (and its Sidebar/Navbar) does NOT re-render. Only the <Outlet /> content changes.
Disadvantages: Can make deeply nested router configurations slightly harder to read at a glance.

⚡ Interview Corner

Q: What happens if a parent route has child routes, but the parent component forgets to include an ``?
A: The child routes will never be rendered on the screen, even if the URL correctly matches the child's path. React Router needs the <Outlet /> component to know exactly where in the parent's DOM tree it should inject the child UI.
