1. Functional Components vs. Class Components

1. Definition

Functional components are simple JavaScript functions that return JSX. Class components are ES6 classes that extend `React.Component` and require a `render()` method.

2. Explanation

  • Historically, Class components were required if you needed to use state or lifecycle methods.
  • With the introduction of Hooks (React 16.8), Functional components gained the ability to use state (useState) and side effects (useEffect).
  • Today, Functional components are the absolute standard. Class components are mostly considered legacy code, except for very specific use cases like Error Boundaries.

3. Example Code

// The Old Way: Class Component
class WelcomeClass extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}</h1>;
  }
}

// The Modern Way: Functional Component
const WelcomeFunction = ({ name }) => {
  return <h1>Hello, {name}</h1>;
};

4. Real-Time Scenarios & Usages

  • Writing 99% of all new UI elements, pages, and layout wrappers in a modern React application.

5. Advantages & Disadvantages

Advantages: Functional components are vastly easier to read, require much less boilerplate code, and completely eliminate confusion surrounding the this keyword.
Disadvantages: None in modern React. However, you must still use Class components if you need to implement componentDidCatch for an Error Boundary.

⚡ Interview Corner

Q: Why did React introduce Hooks and shift away from Classes?
A: Classes made it very difficult to reuse stateful logic between components. Developers had to use complex patterns like "Higher-Order Components" or "Render Props," which led to "wrapper hell" (deeply nested component trees). Hooks allow you to extract and share stateful logic as simple JavaScript functions without changing your component hierarchy.

2. JSX Rules and Best Practices

1. Definition

JSX is a syntax extension that allows writing HTML-like markup inside JavaScript. It comes with strict rules: returning a single parent, closing all tags, and using camelCase for attributes.

2. Explanation

  • Single Parent: A component must return a single root element (or a Fragment <></>).
  • Close Tags: Elements like <img> or <br> must be self-closed like <img />.
  • camelCase: Because it is JS, HTML attributes become camelCase (e.g., onclick becomes onClick, tabindex becomes tabIndex).
  • Reserved Words: Use className instead of class, and htmlFor instead of for.

3. Example Code

const UserProfile = () => {
  const avatarUrl = "/profile.png";
  
  // Using Fragments <>  to avoid adding extra DOM div nodes
  return (
    <>
      <h1 className="title">User Profile</h1>
      {/* Self-closing tag and dynamic JS injection with curly braces */}
      <img src={avatarUrl} alt="User" className="avatar" />
      <label htmlFor="email">Email</label>
    </>
  );
};

4. Real-Time Scenarios & Usages

  • Using curly braces {} to embed dynamic variables, math operations, or function calls directly inside the markup.
  • Using Fragments to group a list of table cells (<td>) without breaking the HTML <table> structure with an unwanted <div>.

5. Advantages & Disadvantages

Advantages: Keeps rendering logic and UI markup coupled in a highly readable format.
Disadvantages: Requires developers to learn slight syntax variations from standard HTML.

⚡ Interview Corner

Q: Why must a React component return a single parent element?
A: Under the hood, JSX compiles into the React.createElement() JavaScript function. A JavaScript function can only return one value (or one object). Returning multiple sibling elements without a parent wrapper would be like trying to return multiple distinct objects from a single function, which is invalid syntax.

3. props (Passing data & `children`)

1. Definition

Props (short for properties) are read-only inputs passed from a parent component down to a child component to configure its rendering or behavior.

2. Explanation

  • Data in React flows unidirectionally (top-down).
  • Props are immutable (read-only). A child component can NEVER modify its own props.
  • Prop Drilling: The process of passing props through multiple layers of intermediate components just to get the data to a deeply nested child.
  • The `children` prop: A special built-in prop that passes whatever is placed between the opening and closing tags of a component.

3. Example Code

// The Child Component using Destructuring and the 'children' prop
const Card = ({ title, children }) => {
  return (
    <div className="card">
      <h2>{title}</h2>
      <div className="card-body">{children}</div>
    </div>
  );
};

// The Parent Component
const App = () => {
  return (
    <Card title="User Settings">
      {/* Everything inside here is passed as the 'children' prop */}
      <p>Change your password here.</p>
      <button>Save</button>
    </Card>
  );
};

4. Real-Time Scenarios & Usages

  • Creating highly reusable UI components (e.g., passing color="red" and text="Delete" to a generic Button component).
  • Using the children prop to create layout wrappers, Modals, or formatting containers.

5. Advantages & Disadvantages

Advantages: Makes components predictable, reusable, and easy to test.
Disadvantages: Prop drilling makes code difficult to maintain if you have to pass data through 5 layers of components. (This is solved by using useContext or Redux).

⚡ Interview Corner

Q: If props are read-only, how does a child component update data in the parent?
A: Through "Inverse Data Flow". The parent must define a state-updating function and pass that function down as a prop to the child. The child then invokes that function (usually via an event like onClick), passing the new data up as an argument.

4. state (Conceptual Overview)

1. Definition

State is a component's internal, mutable memory. When a component's state updates, React automatically triggers a re-render of that component and its children to reflect the new data in the UI.

2. Explanation

  • Unlike props (which are external and read-only), state is internal and controlled entirely by the component itself.
  • State must NEVER be mutated directly (e.g., count = 5). It must be updated using the dedicated setter function provided by React.
  • State updates in React are asynchronous and batched for performance.

3. Example Code

import { useState } from 'react';

const Counter = () => {
  // Array destructuring: [currentValue, setterFunction]
  const [count, setCount] = useState(0);

  const handleIncrement = () => {
    // Correct way to update state
    setCount(count + 1);
  };

  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={handleIncrement}>+1</button>
    </div>
  );
};

4. Real-Time Scenarios & Usages

  • Tracking what the user is typing into a form input.
  • Keeping track of whether a Modal or Dropdown is currently open or closed (isOpen boolean).
  • Storing the JSON data returned from an API fetch.

5. Advantages & Disadvantages

Advantages: Drives dynamic, interactive user interfaces effortlessly.
Disadvantages: Local state gets trapped in the component. If a sibling component needs access to that data, the state must be "lifted up" to their closest common parent, which can lead to prop drilling.

⚡ Interview Corner

Q: What is the primary difference between Props and State?
A: Think of it like a function. Props are the arguments passed into the function from the outside (they are fixed during that render). State is like the local variables declared inside the function that change over time and cause the function to run again (re-render) when modified.

5. Conditional Rendering

1. Definition

Conditional rendering is the ability to output different UI components or elements based on certain JavaScript boolean conditions or state values.

2. Explanation

  • Early Returns (if/else): Good for rendering entirely different screens (like a full page loading spinner).
  • Ternary Operator (`? :`): Best for "if-else" scenarios inside JSX block (e.g., Logged In vs Logged Out).
  • Logical AND (`&&`): Best for "if-only" scenarios where you want to render something ONLY if a condition is true, and nothing otherwise.

3. Example Code

const Dashboard = ({ isLoading, user, hasError }) => {
  // 1. Early Return
  if (hasError) return <h1>Something went wrong!</h1>;
  if (isLoading) return <Spinner />;

  return (
    <div>
      {/* 2. Ternary Operator */}
      {user.isLoggedIn ? <Welcome name={user.name} /> : <LoginButton />}

      {/* 3. Logical AND (&&) */}
      {user.isAdmin && <AdminPanel />}
    </div>
  );
};

4. Real-Time Scenarios & Usages

  • Showing empty states (e.g., "You have no items in your cart").
  • Protecting routes or hiding premium features from free-tier users.

5. Advantages & Disadvantages

Advantages: Clean, declarative way to control the UI.
Disadvantages: Heavily nested ternaries (condition ? (cond2 ? a : b) : c) make JSX unreadable. In such cases, abstract the logic into a separate helper function or component.

⚡ Interview Corner

Q: What happens if you do `{array.length && <List />}` when the array is empty?
A: This is a classic bug. An empty array has a length of 0. In JavaScript, 0 is falsy, so the && short-circuits. However, React renders numbers. So, it will physically print the number "0" onto the screen instead of rendering nothing. Fix: Always use a strict boolean comparison: {array.length > 0 && <List />}.

6. Looping in React (Lists & Keys)

1. Definition

Looping in React is achieved by mapping over an array of data using the JavaScript `.map()` method to return an array of JSX elements. Every dynamically generated element must have a unique `key` prop.

2. Explanation

  • React uses the key prop to uniquely identify elements in a list.
  • Keys are essential for the Reconciliation (Diffing) algorithm. They help React quickly figure out exactly which items were added, changed, or removed.
  • Keys must be unique among siblings and should be stable (like a database ID), not randomly generated during render.

3. Example Code

const TodoList = ({ todos }) => {
  return (
    <ul>
      {todos.map((todo) => (
        // The key is placed on the outermost element returned by the map
        <li key={todo.id} className="todo-item">
          <span>{todo.text}</span>
        </li>
      ))}
    </ul>
  );
};

4. Real-Time Scenarios & Usages

  • Rendering e-commerce product grids, social media feeds, data tables, or navigation menus from an array of JSON data.

5. Advantages & Disadvantages

Advantages: .map() is declarative and returns a new array, keeping rendering logic clean.
Disadvantages: Missing or incorrect keys cause severe performance degradation and bizarre UI state bugs.

⚡ Interview Corner

Q: Why is using the array index as a `key` considered an anti-pattern?
A: If the list is static (never changes), index is fine. But if the list can be reordered, filtered, or items can be added/deleted, the index of an item changes. React uses the key to match state to the component. If the key changes, React might attach the internal state (like text in an input box) to the wrong list item, causing massive data bugs.

7. Event Handling in React (Synthetic Events)

1. Definition

React handles DOM events using a cross-browser wrapper called a `SyntheticEvent`. Events are passed as camelCased props (e.g., `onClick`) and take a function reference rather than a string.

2. Explanation

  • In Vanilla HTML, you write onclick="doSomething()". In React, you write onClick={doSomething}.
  • You must pass the reference to the function, not invoke it immediately.
  • To prevent default browser behavior (like a form refreshing the page), you must explicitly call e.preventDefault() inside your handler.
  • Under the hood, React attaches one massive event listener to the root of the document (Event Delegation) for performance, rather than attaching listeners to every single node.

3. Example Code

const LoginForm = () => {
  const [email, setEmail] = useState('');

  // The 'e' object is a React SyntheticEvent
  const handleSubmit = (e) => {
    e.preventDefault(); // Prevents page reload
    console.log("Submitting:", email);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input 
        type="email" 
        value={email}
        onChange={(e) => setEmail(e.target.value)} // Inline event handler
      />
      <button type="submit">Login</button>
    </form>
  );
};

4. Real-Time Scenarios & Usages

  • Capturing form submissions, tracking button clicks, handling drag-and-drop, or tracking keyboard inputs (onKeyDown).

5. Advantages & Disadvantages

Advantages: Normalizes event behaviors across all browsers. Event delegation makes memory usage extremely efficient.
Disadvantages: In very rare edge cases integrating with vanilla JS libraries outside React, you might need to use native DOM events instead of Synthetic Events.

⚡ Interview Corner

Q: Why do we write `onClick={handleClick}` instead of `onClick={handleClick()}`?
A: handleClick() with parentheses executes the function immediately during the render phase. If that function updates state, it triggers another render, which executes the function again, causing an infinite loop. We pass handleClick (without parentheses) so React only executes it later when the user actually clicks.
