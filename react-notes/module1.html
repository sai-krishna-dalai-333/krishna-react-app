1. let, const, and Scope (Closures in React)

1. Definition

`let` and `const` are ES6 block-scoped variable declarations that replace `var`. A closure is a function's ability to "remember" the variables from its lexical scope even after the outer function has finished executing.

2. Explanation

  • Block-scoped means variables only exist within the { } brackets they are declared in.
  • let allows reassignment; const prevents reassignment of the variable identifier (though objects/arrays held in a const can still be mutated).
  • In React, every functional component render creates a new closure. Hooks like useEffect rely heavily on closures to access the state variables from that specific render cycle.

3. Example Code

function CounterComponent() {
  const [count, setCount] = useState(0); // const used for state reference
  
  const handleDelayedLog = () => {
    // This setTimeout forms a closure over the specific 'count' variable
    setTimeout(() => {
      console.log("Count at time of click:", count);
    }, 2000);
  };
  // ...
}

4. Real-Time Scenarios & Usages

  • Always using const for importing components, declaring configuration objects, and defining state.
  • Using closures inside useEffect cleanup functions to remember the exact ID of a subscription to clear it when unmounting.

5. Advantages & Disadvantages

Advantages: Eliminates variable hoisting bugs associated with var. Promotes immutability.
Disadvantages: Closures in React can become "stale" (referencing old data) if hook dependency arrays are not managed correctly.

⚡ Interview Corner

Q: What is a Stale Closure in React?
A: It occurs when an async function (like setInterval or an event listener) is created inside a component but captures the state from the initial render. When state updates, the closure still looks at the old value. Solution: Use the functional update form of setState (e.g., setCount(prev => prev + 1)) or ensure variables are in the dependency array.

2. Arrow Functions and this context

1. Definition

Arrow functions (`=>`) provide a concise syntax for writing function expressions and do not bind their own `this` context, inheriting it lexically from their parent scope.

2. Explanation

  • They omit the function keyword.
  • If there's only one parameter, parentheses are optional. If one expression, curly braces and return are optional (implicit return).
  • They lack the arguments object.

3. Example Code

// Implicit return, great for React component maps
const UserList = ({ users }) => (
  <ul>
    {users.map(user => <li key={user.id}>{user.name}</li>)}
  </ul>
);

// Arrow function for event handling
const handleClick = (event) => {
  console.log("Button clicked!");
};

4. Real-Time Scenarios & Usages

  • Passing inline callbacks: onClick={() => setIsOpen(true)}.
  • Defining standard React Functional Components.

5. Advantages & Disadvantages

Advantages: Highly readable. Prevents bugs related to losing the this context.
Disadvantages: Cannot be used as constructor functions or methods that require dynamic this binding.

⚡ Interview Corner

Q: Why did we need `bind(this)` in older React Class components, but not with arrow functions?
A: Standard functions create their own this based on how they are invoked. Passing a standard method to onClick invoked it globally, making this undefined. Arrow functions use lexical scoping, permanently locking this to the component instance where they were defined.

3. Template Literals and Object/Array Destructuring

1. Definition

Template literals allow embedded expressions (`${}`) in strings. Destructuring is a syntax to extract values from arrays or properties from objects directly into distinct variables.

2. Explanation

  • Template literals support multi-line strings without ugly concatenations.
  • Object destructuring uses { } and maps by key name. You can alias names and set default values.
  • Array destructuring uses [ ] and maps strictly by position/order.

3. Example Code

// Template Literal
const theme = "dark";
const divClass = `container ${theme === "dark" ? "bg-black text-white" : ""}`;

// Object Destructuring (with default value and alias)
const UserProfile = ({ user }) => {
  const { name, role: userRole = "Guest" } = user;
  return <div>{name} is a {userRole}</div>;
};

// Array Destructuring
const [count, setCount] = useState(0);

4. Real-Time Scenarios & Usages

  • Creating dynamic class names and building API URLs.
  • Cleanly extracting properties from the props object in a component's parameter list.

5. Advantages & Disadvantages

Advantages: Massively reduces boilerplate. Code looks much cleaner.
Disadvantages: Deeply nested object destructuring can cause app crashes if intermediate objects are null/undefined.

⚡ Interview Corner

Q: How does `useState` utilize array destructuring?
A: useState returns an array with exactly two elements: the value and a setter. By using array destructuring, we assign the first item to our chosen variable name and the second to the setter. We use an array instead of an object so the developer can name these freely based on position.

4. Spread (...) and Rest Operators

1. Definition

The Spread operator expands an iterable into its individual elements. The Rest operator collects multiple elements and condenses them into a single array or object.

2. Explanation

  • Spread is used to unpack elements. Crucial for making shallow copies of React state objects to maintain immutability.
  • Rest is used in function definitions to gather remaining arguments, or in destructuring to grab "the rest" of the properties.

3. Example Code

// Spread: Updating State (Immutability)
const [user, setUser] = useState({ name: 'Alex', age: 25, city: 'NY' });
const updateAge = () => {
  // Spread existing user properties, then overwrite age
  setUser(prev => ({ ...prev, age: 26 })); 
};

// Rest: Component Props
const CustomInput = ({ label, ...restProps }) => (
  <div>
    <label>{label}</label>
    <input {...restProps} className="styled-input" />
  </div>
);

4. Real-Time Scenarios & Usages

  • Passing down native HTML attributes to wrapper components without redeclaring them all.
  • Merging two arrays together.

5. Advantages & Disadvantages

Advantages: Crucial for strict state immutability. Clean syntax for merging.
Disadvantages: Only performs a shallow copy. Nested objects still share the same memory reference.

⚡ Interview Corner

Q: Why is spreading an object in React state sometimes not enough?
A: Because ... is a shallow copy. If state is deeply nested, doing { ...data, user: { ...data.user, address: 'LA' } } becomes necessary, which is ugly. Mention alternatives like structuredClone or Immer for nested state.

5. High-Order Array Methods

1. Definition

Array methods like `map`, `filter`, and `reduce` accept a function as an argument (callback) to iterate over, transform, or aggregate data without mutating the original array.

2. Explanation

  • map: Returns a new array of the exact same length, transforming each item.
  • filter: Returns a new array containing only items that pass a test.
  • reduce: Accumulates elements into a single value.

3. Example Code

const users = [
  { id: 1, name: 'Alice', active: true, score: 10 },
  { id: 2, name: 'Bob', active: false, score: 20 },
];

// map (Rendering JSX)
const userCards = users.map(u => <div key={u.id}>{u.name}</div>);

// filter (Conditional Logic)
const activeUsers = users.filter(u => u.active);

// reduce (Aggregation)
const totalScore = users.reduce((acc, curr) => acc + curr.score, 0);

4. Real-Time Scenarios & Usages

  • map is the ONLY way to dynamically render lists of JSX.
  • filter for search bars or removing an item from state.
  • reduce for cart totals.

5. Advantages & Disadvantages

Advantages: Declarative syntax. Preserves immutability.
Disadvantages: Chaining multiple methods iterates over the list multiple times, which can be slightly slower on massive datasets than a single for loop.

⚡ Interview Corner

Q: What happens if you forget to `return` inside a `map` callback?
A: map constructs a new array, but every element will be undefined. In React, it will render nothing and might throw a key error.

6. Promises, async/await, and Error Handling

1. Definition

Promises represent the eventual completion of an async operation. `async/await` is syntactic sugar over Promises making code look synchronous. `try/catch` catches errors.

2. Explanation

  • A Promise is Pending, Fulfilled, or Rejected.
  • Marking a function async automatically returns a Promise.
  • await pauses execution until the Promise settles.

3. Example Code

const fetchUserData = async (userId) => {
  try {
    const response = await fetch(`https://api.com/users/${userId}`);
    if (!response.ok) throw new Error("Network response was not ok");
    
    const data = await response.json();
    return data;
  } catch (error) {
    console.error("Failed to fetch user:", error.message);
  }
};

4. Real-Time Scenarios & Usages

  • Used inside useEffect to fetch data on mount.
  • Submitting form data to a server.

5. Advantages & Disadvantages

Advantages: Prevents "callback hell" (.then().catch()).
Disadvantages: You cannot use await directly inside a top-level React component; must be inside a useEffect.

⚡ Interview Corner

Q: Can you pass an `async` function directly to `useEffect`?
A: No. useEffect expects its callback to return either nothing or a cleanup function. An async function always returns a Promise. You must define the async function inside the useEffect and call it immediately.

7. Modules (import/export)

1. Definition

ES6 Modules split JS code into files. `export` makes functions/objects available outside; `import` brings them in.

2. Explanation

  • Named Exports: Export multiple items. Must import with exact names inside {}.
  • Default Exports: Only ONE per file. You can name it whatever you want upon import.

3. Example Code

// --- utils.js ---
export const formatDate = (date) => { ... }; // Named
const calculateTax = (amount) => { ... };
export default calculateTax; // Default

// --- App.jsx ---
import TaxCalculator, { formatDate } from './utils';

4. Real-Time Scenarios & Usages

  • Every React component is typically built in its own file and exported.

5. Advantages & Disadvantages

Advantages: Code splitting, reusability, scoping.
Disadvantages: Can lead to messy import headers if folder structure is poor.

⚡ Interview Corner

Q: When would you prefer Named exports over Default exports?
A: Named exports enforce consistent naming across the codebase and provide better IDE auto-completion. Default exports can be accidentally imported with different names, causing confusion.

8. Short-circuit evaluation & Optional Chaining

1. Definition

Logical operators evaluate conditions safely. `&&`, `||`, `??` (Nullish Coalescing), and `?.` (Optional Chaining) prevent crashes on missing data.

2. Explanation

  • &&: Returns B if A is truthy (conditional rendering).
  • ||: Returns B if A is falsy (fallbacks).
  • ??: Returns B only if A is null or undefined.
  • ?.: Stops evaluating and returns undefined if object is missing.

3. Example Code

const UserDashboard = ({ user, isLoading }) => {
  return (
    <div>
      {/* 1. Conditional Rendering */}
      {isLoading && <Spinner />}

      {/* 2. Optional Chaining and Fallback */}
      <h1>Welcome, {user?.profile?.firstName || "Guest"}</h1>

      {/* 3. Nullish Coalescing - Allows '0' to be valid */}
      <p>Account Balance: {user?.balance ?? 'N/A'}</p>
    </div>
  );
}

4. Real-Time Scenarios & Usages

  • Waiting for API data to load safely (e.g., data?.map()).

5. Advantages & Disadvantages

Advantages: Reduces verbose if/else blocks inside JSX.
Disadvantages: Overusing && can sometimes accidentally render a 0 on the screen if the condition evaluates to the number 0.

⚡ Interview Corner

Q: Difference between `||` and `??`?
A: || checks for any falsy value (0, "", false). ?? checks only for null/undefined. If state is 0, count || 10 evaluates to 10. count ?? 10 evaluates to 0.
