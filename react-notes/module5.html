1. Folder Structure & Clean Architecture Best Practices

1. Definition

Clean architecture in React is the practice of organizing code logically—often grouped by feature or domain rather than file type—to make the codebase scalable, maintainable, and predictable for large teams.

2. Explanation

  • File-type Organization (Legacy): Grouping all components in one folder, all hooks in another. This becomes unmanageable as the app grows.
  • Feature-based Organization (Modern): Grouping code by its domain (e.g., /features/auth, /features/cart). Everything related to auth (components, hooks, API calls) lives together.
  • Shared, highly reusable pieces (like a generic Button component or utility functions) live in a global /shared or /common directory.

3. Example Code

src/
├── assets/         # Global images, fonts
├── components/     # Global/Shared UI (Button, Input, Layout)
├── features/       # Feature-specific modules
│   ├── auth/       # Domain: Authentication
│   │   ├── components/  # LoginForm, RegisterForm
│   │   ├── hooks/       # useAuth, useLogin
│   │   ├── api/         # login(), logout()
│   │   └── index.js     # Public API for this feature
│   └── cart/       # Domain: Shopping Cart
├── hooks/          # Global hooks (useWindowSize, useTheme)
├── utils/          # Global helpers (formatDate, calculateTax)
└── App.jsx         # Root component

4. Real-Time Scenarios & Usages

  • Enterprise applications with multiple teams. Team A can work exclusively inside features/auth without colliding with Team B working in features/billing.

5. Advantages & Disadvantages

Advantages: Code is highly discoverable. Deleting a feature is as simple as deleting one folder. Reduces merge conflicts.
Disadvantages: Requires strict discipline to enforce. Developers might accidentally create cross-feature dependencies if not careful.

⚡ Interview Corner

Q: What is a "Barrel File" (index.js) in feature-based architecture?
A: A barrel file is an index.js at the root of a feature folder that exports ONLY the components or hooks that other parts of the app are allowed to use. It acts as a public API for that feature, hiding internal helper functions and enforcing strict encapsulation.

2. Controlled vs. Uncontrolled Components (Forms)

1. Definition

A Controlled Component is an input where React state acts as the "single source of truth". An Uncontrolled Component is an input where the DOM itself maintains its own state, which React accesses via a `ref`.

2. Explanation

  • Controlled: Uses value={state} and onChange={e => setState(e.target.value)}. React knows the exact value on every single keystroke.
  • Uncontrolled: Uses a useRef to target the DOM node directly. You only read the value when you actually need it (e.g., when the user clicks "Submit"). Uses defaultValue for initial state.

3. Example Code

// 1. Controlled Component (React manages state)
const ControlledForm = () => {
  const [email, setEmail] = useState('');
  return (
    <input 
      value={email} 
      onChange={(e) => setEmail(e.target.value)} 
    />
  );
};

// 2. Uncontrolled Component (DOM manages state)
const UncontrolledForm = () => {
  const emailRef = useRef(null);
  const handleSubmit = () => alert(emailRef.current.value);
  
  return (
    <>
      <input ref={emailRef} defaultValue="test@test.com" />
      <button onClick={handleSubmit}>Submit</button>
    </>
  );
};

4. Real-Time Scenarios & Usages

  • Controlled: Instant form validation (e.g., password strength meter), dynamically disabling submit buttons, or searching a list as the user types.
  • Uncontrolled: Integrating React with non-React libraries (like a legacy jQuery plugin) or building very simple, massive forms where you don't want re-renders on every keystroke.

5. Advantages & Disadvantages

Controlled Adv: Highly predictable and powerful for dynamic UI updates. Disadv: Re-renders the component on every single keystroke.
Uncontrolled Adv: High performance (no constant re-renders). Disadv: Hard to do live validation. (Note: React Hook Form solves this by combining the best of both worlds).

⚡ Interview Corner

Q: Why does React show a warning: "You provided a `value` prop to a form field without an `onChange` handler"?
A: If you provide a value prop without an onChange handler, you are creating a Controlled component but giving it no way to update. React locks the input, making it strictly read-only. The user won't be able to type anything.

3. Higher-Order Components (HOCs) vs. Custom Hooks

1. Definition

A Higher-Order Component (HOC) is a function that takes a component and returns a new component, injecting additional props or logic. A Custom Hook is a function that extracts stateful logic to be reused inside functional components.

2. Explanation

  • Before Hooks existed (React < 16.8), HOCs were the primary way to share logic between Class components (e.g., Redux's connect() or Router's withRouter()).
  • HOCs literally wrap your component in another component, augmenting it with new props.
  • Custom Hooks completely replaced HOCs for sharing stateful logic because they don't require altering the component hierarchy.

3. Example Code

// 1. The Old Way: HOC (Wrapper)
const withAuth = (WrappedComponent) => {
  return (props) => {
    const isAuth = checkAuth();
    if (!isAuth) return <Redirect to="/login" />;
    return <WrappedComponent {...props} user={getUser()} />;
  };
};
const ProtectedProfile = withAuth(UserProfile);

// 2. The Modern Way: Custom Hook
const UserProfile = () => {
  const { user, isAuth } = useAuth(); // No wrappers needed!
  if (!isAuth) return <Redirect to="/login" />;
  return <div>Welcome, {user.name}</div>;
};

4. Real-Time Scenarios & Usages

  • Hooks: Used for 95% of logic sharing (fetching, themes, auth state, window dimensions).
  • HOCs: Still occasionally used for global cross-cutting concerns that strictly require wrapping the component, such as Error Boundaries (withErrorBoundary) or Performance Tracking wrappers.

5. Advantages & Disadvantages

HOC Disadvantages: Causes "Wrapper Hell" (deeply nested component trees in DevTools). Leads to prop name collisions (two HOCs injecting a prop named `data`).
Hooks Advantages: Flattens the component tree. Clearer origin of data (you can see exactly where user came from: useAuth()).

⚡ Interview Corner

Q: Are HOCs completely dead in modern React?
A: No. While Custom Hooks replaced them for logic sharing, HOCs are still useful when you need to intercept rendering logic from the outside. For example, React.memo() is technically a Higher-Order Component because it wraps a component to control its re-render behavior based on props.

4. Error Boundaries (Handling app crashes gracefully)

1. Definition

Error Boundaries are React components that catch JavaScript errors anywhere in their child component tree, log those errors, and display a fallback UI instead of crashing the entire application.

2. Explanation

  • Currently, there is no Functional Component/Hook equivalent for Error Boundaries. They MUST be written as Class components.
  • They rely on two lifecycle methods: static getDerivedStateFromError() (to render fallback UI) and componentDidCatch() (to log the error to a service like Sentry).
  • They only catch errors during rendering, in lifecycle methods, and in constructors of the whole tree below them.

3. Example Code

class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false };
  }

  // 1. Update state so the next render shows the fallback UI
  static getDerivedStateFromError(error) {
    return { hasError: true };
  }

  // 2. Log the error to an error reporting service
  componentDidCatch(error, errorInfo) {
    logErrorToMyService(error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      return <h1>Something went wrong in this widget.</h1>;
    }
    return this.props.children; 
  }
}

// Usage:
// <ErrorBoundary> <ComplexChartWidget /> </ErrorBoundary>

4. Real-Time Scenarios & Usages

  • Wrapping the root <App /> component to prevent the "White Screen of Death" for users.
  • Wrapping individual, unreliable widgets (like a 3rd party charting library). If the chart crashes, the rest of the dashboard remains fully functional.

5. Advantages & Disadvantages

Advantages: Prevents total app failure and provides graceful degradation.
Disadvantages: Must be written as a Class component. (Most modern developers use the popular react-error-boundary npm package to abstract this class away).

⚡ Interview Corner

Q: Will an Error Boundary catch an error thrown inside a `setTimeout` or a `fetch` Promise?
A: No! Error Boundaries do NOT catch errors in asynchronous code (Promises, setTimeout), event handlers (onClick), or Server-side rendering. They only catch errors that happen during the synchronous React rendering phase. Async errors must be caught using standard try/catch blocks.

5. Portals (ReactDOM.createPortal)

1. Definition

React Portals provide a first-class way to render children into a DOM node that exists completely outside the DOM hierarchy of the parent component.

2. Explanation

  • Normally, a component's render output is placed in the DOM physically inside its parent <div>.
  • This causes massive CSS issues for Modals and Tooltips. If a parent has overflow: hidden or a strict z-index, the Modal gets trapped inside the parent and cut off.
  • createPortal solves this by physically teleporting the HTML of the child to a different part of the actual index.html document (usually directly under the <body>).

3. Example Code

import { createPortal } from 'react-dom';

const Modal = ({ isOpen, children }) => {
  if (!isOpen) return null;

  // We are rendering 'children', but appending them directly to document.body
  // (or a dedicated div like document.getElementById('modal-root'))
  return createPortal(
    <div className="modal-overlay">
      <div className="modal-content">
        {children}
      </div>
    </div>,
    document.body // Target DOM node
  );
};

// Usage deeply nested inside some component:
// <Modal isOpen={true}> <h1>I float above everything!</h1> </Modal>

4. Real-Time Scenarios & Usages

  • Dialog boxes / Modals.
  • Hover Tooltips and complex Dropdown menus.
  • Global Toast notification snackbars.

5. Advantages & Disadvantages

Advantages: Completely bypasses tricky CSS stacking contexts (z-index) and overflow restrictions of parent containers.
Disadvantages: Requires managing DOM nodes slightly outside of the standard React flow (e.g., ensuring the target div actually exists in the HTML file).

⚡ Interview Corner

Q: If a Portal renders its HTML completely outside the parent DOM node, how does Event Bubbling work?
A: This is the magic of Portals! Even though the physical HTML is appended to the <body>, the component still exists in the React component tree exactly where you placed it. Because React uses its own Synthetic Event system, an onClick inside the portal will still bubble up through the React ancestors, even though it wouldn't bubble up through the real DOM ancestors.
