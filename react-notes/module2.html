1. What is React? (Declarative vs. Imperative, SPA concept)

1. Definition

React is an open-source JavaScript library for building user interfaces. It uses a declarative paradigm and is primarily used to build Single Page Applications (SPAs).

2. Explanation

  • Declarative: You tell React what the UI should look like based on the current state. React automatically figures out how to update the DOM. This is the opposite of Imperative (Vanilla JS), where you must write step-by-step instructions to select and mutate elements.
  • Component-Based: UIs are broken down into isolated, reusable pieces of code (components).
  • SPA (Single Page Application): The server sends a single, blank HTML file to the browser. React then takes over, dynamically rendering and swapping out content using JavaScript without ever needing a full page refresh.

3. Example Code

// Imperative (Vanilla JS - Step-by-Step)
const btn = document.createElement('button');
btn.innerText = 'Click Me';
btn.addEventListener('click', () => btn.classList.add('active'));
document.getElementById('root').appendChild(btn);

// Declarative (React - State Driven)
const Button = () => {
  const [isActive, setIsActive] = useState(false);
  // We simply declare WHAT it should look like based on 'isActive'
  return (
    <button 
      className={isActive ? 'active' : ''} 
      onClick={() => setIsActive(true)}
    >
      Click Me
    </button>
  );
};

4. Real-Time Scenarios & Usages

  • Building complex dashboards (like Jira or Facebook) where multiple UI components need to react to the same underlying data changes without page reloads.

5. Advantages & Disadvantages

Advantages: Fluid, app-like user experience; massive code reusability; easier to debug state-driven UI.
Disadvantages: Initial load times can be slower (downloading the JS bundle). Out-of-the-box SEO is poor for SPAs (requiring solutions like Next.js).

⚡ Interview Corner

Q: What is the difference between a Library and a Framework? (Why is React a library?)
A: React is strictly a UI library. It only handles the view layer (rendering HTML based on state). It does not include built-in routing, HTTP clients, or form validation. A framework (like Angular) is a complete ecosystem that dictates how you structure your entire application and provides built-in tools for routing, fetching, and state.

2. How React Compiles: JSX and Babel/Vite

1. Definition

JSX (JavaScript XML) is a syntax extension that looks like HTML but lives inside JavaScript. Because browsers cannot read JSX natively, tools like Babel or Vite's esbuild compile it down to regular JavaScript function calls.

2. Explanation

  • JSX is purely syntactic sugar. It makes writing React components intuitive.
  • Historically, a tool called Babel was used to transpile (translate) JSX into React.createElement().
  • Modern tooling (like Vite) uses faster compilers written in Go or Rust (like esbuild or SWC) to do this transformation in milliseconds.
  • Since React 17, the new JSX transform automatically imports the necessary compilation functions under the hood, so you no longer need import React from 'react' in every file.

3. Example Code

// What you write (JSX):
const heading = <h1 className="title">Hello World</h1>;

// What the compiler (Babel/Vite) turns it into (Vanilla JS):
// Older React (Pre-v17):
const heading = React.createElement('h1', { className: 'title' }, 'Hello World');

// Modern React (Post-v17 JSX Transform):
import { jsx as _jsx } from "react/jsx-runtime";
const heading = _jsx("h1", { className: "title", children: "Hello World" });

4. Real-Time Scenarios & Usages

  • Any time you write HTML-like tags inside a `.jsx` or `.tsx` file, you are relying on this compilation process.

5. Advantages & Disadvantages

Advantages: Visual clarity. You can embed raw JS logic directly into UI markup using curly braces {}.
Disadvantages: Requires a build step (webpack/Vite). You cannot just drop a React file into an HTML script tag without compilation (unless using a heavy CDN babel-standalone).

⚡ Interview Corner

Q: Why do we use `className` instead of `class` in JSX?
A: Because JSX is compiled into JavaScript. class is a reserved keyword in JavaScript (used for creating ES6 classes). Therefore, React chose the DOM API property name className to define HTML classes to avoid syntax errors during compilation.

3. The Virtual DOM & The Reconciliation Algorithm (Diffing)

1. Definition

The Virtual DOM (VDOM) is a lightweight JavaScript object representation of the actual actual DOM. Reconciliation is the process where React compares the new VDOM with the old VDOM (Diffing) to determine the absolute minimum number of DOM updates required.

2. Explanation

  • The Problem: Updating the Real DOM directly (repainting the screen, recalculating CSS) is the slowest operation in web development.
  • The Solution: When state changes, React creates a completely new Virtual DOM tree.
  • React then compares (diffs) this new tree against the previous Virtual DOM tree.
  • It calculates exactly what changed (e.g., "only the text inside this one span changed").
  • It then updates the Real DOM with only those specific changes in a single, efficient batch operation.

3. Example Code

// Conceptual representation of a Virtual DOM object
const vdomNode = {
  type: 'div',
  props: { className: 'container' },
  children: [
    { type: 'h1', props: {}, children: 'Hello' },
    // If state updates 'count', only this specific text node changes in the VDOM object
    { type: 'p', props: {}, children: `Count: ${count}` } 
  ]
};

4. Real-Time Scenarios & Usages

  • Typing in a text input inside a massive form. React will only update the specific input field in the real DOM, without re-rendering the hundreds of other elements around it.

5. Advantages & Disadvantages

Advantages: Massive performance gains by avoiding expensive, unnecessary real DOM repaints.
Disadvantages: Keeping two copies of the UI tree in memory (the current VDOM and the new VDOM) consumes RAM. (Newer frameworks like Svelte avoid VDOM entirely via ahead-of-time compilation).

⚡ Interview Corner

Q: How does React make Diffing an O(n) operation instead of O(n³)?
A: Comparing two trees is traditionally very slow. React uses a heuristic algorithm based on two assumptions:
1. Two elements of different types (e.g., a <div> turning into an <span>) will produce different trees, so React throws away the old one and rebuilds it completely.
2. Developers use the key prop to hint to React which child elements are stable across renders, preventing unnecessary unmounting and remounting of lists.

4. React Fiber Architecture

1. Definition

React Fiber is the core internal rendering engine introduced in React 16. Its primary superpower is "incremental rendering"—the ability to split rendering work into chunks and spread it out over multiple frames.

2. Explanation

  • Before Fiber (the old Stack Reconciler), when React started rendering a component tree, it could not be stopped. If the tree was huge, the main thread was blocked, causing animations to drop frames (jank).
  • Fiber breaks the work into units called "fibers".
  • It acts as a scheduler. It can assign priorities to different updates.
  • For example, typing in an input (user interaction) is high priority. Fetching data in the background is low priority. Fiber can pause the low-priority render, handle the high-priority typing, and then resume the background render.

3. Example Code

// While you rarely touch Fiber directly, React 18 hooks expose its power.
import { useTransition } from 'react';

const SearchComponent = () => {
  const [isPending, startTransition] = useTransition();
  const [query, setQuery] = useState('');

  const handleChange = (e) => {
    // High Priority: Update the input field immediately
    setQuery(e.target.value); 
    
    // Low Priority: Fiber will pause this heavy list filtering if the user types again
    startTransition(() => {
      filterMassiveDataList(e.target.value); 
    });
  };
  // ...
};

4. Real-Time Scenarios & Usages

  • Enables Concurrent Mode features like Suspense, useTransition, and useDeferredValue.
  • Keeps the application highly responsive even when rendering massive, CPU-intensive data tables.

5. Advantages & Disadvantages

Advantages: Non-blocking UI, smoother animations, prioritizing urgent user inputs over background tasks.
Disadvantages: Highly complex source code. Developers don't interact with it directly, but bugs in concurrent UI logic can be hard to trace.

⚡ Interview Corner

Q: What does it mean that React rendering is "interruptible"?
A: Thanks to Fiber, React no longer processes the entire Virtual DOM tree synchronously in one go. If a more urgent task comes into the browser (like an animation frame or a user click), React can pause its rendering work, yield control back to the browser to handle the event, and then pick up the rendering work where it left off.

5. Component Lifecycle (Class to Functional Shift)

1. Definition

The Component Lifecycle refers to the phases a component goes through: Mounting (creation), Updating (re-rendering), and Unmounting (deletion). Historically managed via specific Class methods, modern React handles this via Functional components and the useEffect hook.

2. Explanation

  • Mounting: The component is placed into the DOM for the first time.
  • Updating: The component re-renders because its state or props changed.
  • Unmounting: The component is removed from the DOM entirely.
  • In Class components, you used methods like componentDidMount. In Functional components, you use useEffect to achieve the same timing.

3. Example Code

import { useEffect } from 'react';

const FunctionalLifecycle = ({ id }) => {

  useEffect(() => {
    // 1. MOUNTING Phase (and UPDATING if 'id' changes)
    console.log(`Setting up subscription for id: ${id}`);
    const connection = createConnection(id);
    connection.connect();

    // 2. UNMOUNTING Phase (Cleanup function)
    return () => {
      console.log(`Cleaning up subscription for id: ${id}`);
      connection.disconnect();
    };
    
  }, [id]); // Dependency array dictates when the UPDATE phase occurs

  return <div>Listening to {id}</div>;
};

4. Real-Time Scenarios & Usages

  • Mount: Fetching initial user data from an API when a dashboard loads.
  • Update: Refetching that data when the user selects a different date range.
  • Unmount: Removing event listeners (like scroll or resize tracking) when navigating to a different page to prevent memory leaks.

5. Advantages & Disadvantages

Advantages of Functional Shift: It co-locates related logic. In Class components, setup was in DidMount and teardown in WillUnmount. Now, setup and teardown for a specific feature exist inside the exact same useEffect.
Disadvantages: The mental model of useEffect dependencies can be challenging to grasp compared to explicit lifecycle methods.

⚡ Interview Corner

Q: Exactly when does the cleanup function (the return statement inside useEffect) run?
A: It runs at two distinct times:
1. Just before the component unmounts from the screen (classic teardown).
2. Before the next effect runs on a re-render. If a dependency changes, React first runs the cleanup function from the previous render to wipe the slate clean, and then runs the new setup function.
